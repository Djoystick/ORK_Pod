# Phase 15 — Структурирование импортированных описаний и UTF-8 cleanup

## 1) Стартовое состояние (фактически)
- Базовый ingestion/live sync после Phase 14B работал.
- На detail-страницах импортированный YouTube-текст визуально выглядел как «сплошная стена».
- В админке оставались видимые mojibake-строки в части контентных форм/фильтров.

## 2) Корневая причина рендера «одним блоком»
1. В ingestion-нормализации описание проходит через `normalizeText(stripHtml(...))`, что исторически приводит к сплющиванию переносов строк в одномерный текст.
2. В detail view описание рендерилось одним `<p>{item.description}</p>` без отдельного presentation-слоя для импортированных источников.

Итог: даже при полезной структуре исходного описания (ссылки, таймкоды, списки) пользователь видел тяжёлый неструктурированный блок.

## 3) Что изменено в presentation-слое

### 3.1 Новый компонент структурированного вывода
- Добавлен `src/components/archive/imported-description.tsx`.
- Компонент:
  - нормализует переносы строк безопасно для отображения;
  - делит текст на блоки (абзацы / list-like / link-heavy);
  - linkify-рендерит URL внутри текста;
  - выводит URL-only блоки отдельным компактным списком «Ссылки из описания».

### 3.2 Интеграция в detail page
- Обновлён `src/app/streams/[slug]/page.tsx`.
- Для `item.sourceType === "imported"` теперь используется `ImportedDescription` (с приоритетом `item.body`, fallback на `item.description`).
- Для неимпортированного контента прежний рендер сохранён.

## 4) UX длинных описаний
- Добавлен безопасный режим сворачивания длинного текста:
  - порог по объёму блоков/символов;
  - кнопка `Показать полностью / Свернуть описание`;
  - компактный первый экран + доступ к полному тексту без потери данных.

## 5) Ссылки / promo-блоки / сырой источник
- URL остаются кликабельными.
- URL-heavy фрагменты выводятся визуально спокойнее, чем «сырой» монолит.
- Исходные ссылки не удаляются.
- Сырой импортированный source-текст на уровне хранения не перезаписывался и не заменялся выдуманными summary.

## 6) UTF-8 / mojibake cleanup
- Исправлены битые строки в:
  - `src/app/admin/content/page.tsx`
  - `src/app/admin/content/[id]/edit-content-form.tsx`
- Текстовые метки/фильтры/кнопки возвращены к корректному UTF-8 (русский).
- Дополнительно проверены related admin surfaces; критичных новых mojibake-строк в рамках этого прохода не выявлено.

## 7) Безопасность и non-regression
- Ingestion-архитектура и sync-пайплайн не менялись.
- Дедупликация/anti-concurrency/review safety не ослаблялись.
- Изменения локализованы в UI-представлении detail/admin.
